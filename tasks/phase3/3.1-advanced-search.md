# Task 3.1 — Advanced Search

**Phase:** 3 — Advanced Features
**Status:** [x] Complete

## Objective
Implement multi-criteria document search using PostgreSQL full-text search, with the architecture ready for optional Elasticsearch upgrade.

## Steps

### PostgreSQL Full-Text Search
- [x] Add migration: `search_vector` column to `documents` (`tsvector` type)
  - Value: `to_tsvector('english', title || ' ' || document_type_name || ' ' || student_name)` — populated by trigger on insert/update
- [x] Add migration: `CREATE INDEX idx_documents_fts ON documents USING GIN(search_vector)`
- [x] Add `search_vector @@ plainto_tsquery('english', ?)` to search query when `q` param is present

### Search Service (`documentService.js` addition — `search(filters, actor)`)
- [x] Accepted filter params:
  - `q` — keyword (full-text)
  - `type` — document_type_id
  - `status` — one or comma-separated list
  - `from` / `to` — date range on `created_at`
  - `student_id` — exact match (admin/finance_staff only)
  - `amount_min` / `amount_max` — financial_amount range
  - `department_id` — department filter
  - `page`, `limit` — pagination (default: page 1, limit 20, max limit 100)
- [x] All filters are optional and combined with AND logic
- [x] Role scoping applied: student can only search own documents

### Search Controller & Route
- [x] `GET /api/v1/documents/search` — all params as query string
- [x] Response: `{ success: true, data: [], meta: { page, limit, total, totalPages, filters } }`

### Storage Adapter Interface (for future Elasticsearch)
- [x] Extract search logic into `services/searchService.js` with a single `search(filters, actor)` method
- [x] Current implementation: delegates to `documentService.search()` (PostgreSQL FTS)
- [x] Future implementation: swap to `elasticService.search()` by changing one line in `searchService.js`

### Frontend (`DocumentListComponent` update)
- [x] Search bar at top of document list (debounced 300ms — use `debounceTime` RxJS operator on a `FormControl`)
- [x] Filter sidebar/panel:
  - Status checkboxes
  - Document type dropdown
  - Date range picker (native `<input type="date">`)
  - Financial amount range (min/max number inputs)
- [x] Active filters shown as dismissible tags below search bar
- [x] "Clear all filters" button
- [x] URL search params sync: filters reflected in URL via `Router` and `ActivatedRoute` so they survive page refresh and can be shared

### Tests
- [x] Unit: search service builds correct SQL with each filter combination
- [x] Unit: student role cannot bypass scoping even with explicit `student_id` param pointing to another student
- [x] Integration: seed 10 documents with varied metadata, run 5 filter combinations, verify correct subsets returned

## Acceptance Criteria
- Keyword search returns documents matching title or document type
- All filter combinations work correctly and are combinable
- Role scoping is preserved in all search results
- Search responds in under 500ms for up to 10,000 documents
- URL reflects current search state (shareable link)
