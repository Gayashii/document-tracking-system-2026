# Task 3.3 — Audit Trail UI

**Phase:** 3 — Advanced Features
**Status:** [x] Complete

## Objective
Surface the immutable audit log to admins and auditors via a filterable, paginated UI.

## Steps

### Audit Service (`services/auditService.js`)
- [x] `query(filters)`: paginated query on `audit_logs` table
  - Filter params: `actor_id`, `entity_type`, `entity_id`, `action`, `from`, `to`
  - Always sorted by `created_at DESC`
  - Join `users` to resolve `actor_id` → `actor_name`, `actor_email`, `actor_role`
- [x] `getEntityHistory(entityType, entityId)`: all audit entries for a specific document or user

### Controller & Route (`audit.routes.js`)
- [x] `GET /api/v1/audit` — roles: admin, auditor — paginated audit log with filters
- [x] `GET /api/v1/audit/entity/:type/:id` — roles: admin, auditor — all events for one entity

### Frontend (`src/app/features/reports/AuditTrailComponent`)
- [x] Filterable, sortable table:
  - Columns: Timestamp, Actor (name + role), Action, Entity Type, Entity ID, IP Address, Details
  - Filters: actor name/email, action type, entity type, date range
- [x] "Details" column: collapsed by default; click to expand and show full metadata JSON in a formatted code block
- [x] Pagination: server-side
- [x] Export audit log: CSV download button (uses existing export service)
- [x] Link from `DocumentDetail.jsx`: "View Audit History" → `AuditTrail` pre-filtered to that document

### Audit Log Actions Reference (document in `constants/auditActions.js`)
- [x] Define all action constants used across the system:
  - `document.upload`, `document.status_changed`, `document.deleted`, `document.version_created`
  - `user.login`, `user.logout`, `user.login_failed`, `user.mfa_enabled`, `user.password_reset`
  - `document.barcode_set`
  - `report.exported`, `admin.user_created`, `admin.user_deactivated`

### Tests
- [x] Unit: audit query applies all filter combinations correctly
- [x] Integration: perform 5 actions across the system, query audit log, verify all 5 appear with correct metadata

## Acceptance Criteria
- Admins and auditors can filter audit logs by actor, action, entity, and date range
- Each audit entry shows actor details (name, role, IP address)
- Metadata JSON is readable (formatted, not raw string)
- The audit log cannot be modified through any UI action
- Audit trail is accessible from document detail page pre-filtered to that document

## Dependencies
- Task 1.2 (audit_logs table, append-only), Task 2.5 (RBAC), Task 3.2 (export service for CSV)
