# Task 3.5 — Frontend Polish

**Phase:** 3 — Advanced Features
**Status:** [ ] Not Started

## Objective
Improve frontend UX with loading states, optimistic updates, lazy loading, and a consistent component library.

## Steps

### Loading States
- [ ] Replace all spinners on document list and detail pages with skeleton screens (CSS animated placeholders matching the layout shape)
- [ ] `SkeletonComponent`: configurable width, height, border-radius, with shimmer animation

### Optimistic UI Updates
- [ ] `DocumentActionsService`: encapsulates status transition calls
  - On action: immediately update local `BehaviorSubject` document state to new status
  - On API error: revert to previous status + show error toast
  - Keeps page components thin (no direct API calls in components)
- [ ] Apply to: status changes in `DocumentDetailComponent` and `DocumentListComponent` quick-actions

### Code Splitting (Lazy Loading)
- [ ] All feature modules are lazy-loaded via Angular Router (`loadComponent` / `loadChildren`)
- [ ] Separate lazy chunks for: auth routes, document routes, admin routes, reports routes
- [ ] Run `source-map-explorer dist/client/browser/*.js` on production build; ensure initial bundle is under 200 KB gzipped

### Accessibility
- [ ] All form inputs have associated `<label>` elements or `aria-label`
- [ ] Status badges have `aria-label` describing the status
- [ ] Modals trap focus and close on Escape key (use Angular CDK `FocusTrap`)
- [ ] All interactive elements are keyboard navigable
- [ ] Run `axe` accessibility checks in CI

### Print Styles
- [ ] Global `@media print` CSS: hide sidebar, topbar, action buttons, filters

### Error Handling
- [ ] Global Angular `ErrorHandler` override: catches unhandled errors, shows a friendly "Something went wrong" message with a reload button
- [ ] Log caught errors to `console.error` (or a future error tracking service)

### Empty States
- [ ] `EmptyStateComponent`: icon + heading + subtext, used when lists return 0 results
- [ ] Contextual empty states: "No documents submitted yet" for students, "No pending documents" for finance staff queue

### Tests
- [ ] Unit: `SkeletonComponent` renders with correct dimensions
- [ ] Unit: `DocumentActionsService` reverts state on API error
- [ ] Unit: custom `ErrorHandler` catches a thrown error and triggers fallback UI
- [ ] Lighthouse score: Performance ≥ 80, Accessibility ≥ 90

## Acceptance Criteria
- No visible spinner on any page — only skeleton screens during loading
- Status change feels instant (optimistic update) and correctly reverts on error
- Initial page load bundle under 200 KB gzipped
- All pages pass keyboard navigation testing
- Lighthouse accessibility score ≥ 90

## Dependencies
- Task 1.6 (frontend scaffold), Task 2.7 (workflow UI), Task 3.1 (search UI), Task 3.2 (reports UI)
