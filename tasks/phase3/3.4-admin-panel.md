# Task 3.4 — Admin Panel Completion

**Phase:** 3 — Advanced Features
**Status:** [ ] Not Started

## Objective
Build the full admin interface for managing users, departments, document types, and system settings.

## Steps

### User Management
- [ ] `GET /api/v1/users` — paginated user list with search by name/email, filter by role and status
- [ ] `POST /api/v1/users` — create any user (admin creates finance_staff/auditor accounts)
- [ ] `PATCH /api/v1/users/:id` — update role, is_active
- [ ] `DELETE /api/v1/users/:id` — deactivate (set `is_active=false`), never hard-delete; revoke all refresh tokens
- [ ] `POST /api/v1/users/:id/mfa/reset` — force MFA re-enrolment (clears `mfa_secret`, sets `mfa_enabled=false`)
- [ ] All user mutations logged to `audit_logs`

### Department Management
- [ ] `GET /api/v1/departments` — list all
- [ ] `POST /api/v1/departments` — create
- [ ] `PATCH /api/v1/departments/:id` — update name/code
- [ ] `DELETE /api/v1/departments/:id` — deactivate (cannot delete if documents exist in that department)

### Document Type Management
- [ ] `GET /api/v1/document-types` — list all
- [ ] `POST /api/v1/document-types` — create with `requires_approval` flag
- [ ] `PATCH /api/v1/document-types/:id` — update
- [ ] `DELETE /api/v1/document-types/:id` — deactivate (cannot delete if documents exist of that type)

### System Settings
- [ ] `GET /api/v1/settings` — return all settings as key-value object
- [ ] `PATCH /api/v1/settings` — update one or more settings (admin only)
- [ ] Configurable settings (stored in `system_settings` table):
  - `max_file_size_mb` (default: 10)
  - `allowed_mime_types` (default: `application/pdf,image/jpeg,image/png`)
  - `session_timeout_minutes` (default: 60)
  - `overdue_threshold_days` (default: 5)
  - `mfa_required_roles` (default: `admin,finance_staff`)
  - `email_notifications_enabled` (default: true)

### Frontend (`src/app/features/admin/`)
- [ ] `UserManagementComponent`: searchable table, create user modal, edit role dropdown, deactivate button, force-reset MFA button
- [ ] `DepartmentSettingsComponent`: simple CRUD table with inline edit
- [ ] `DocumentTypeSettingsComponent`: CRUD table, `requires_approval` toggle
- [ ] `SystemSettingsComponent`: settings reactive form with save button; shows last updated by + timestamp for each setting

### Tests
- [ ] Integration: create user → login as that user → deactivate → verify login fails
- [ ] Integration: force MFA reset → verify user must re-enrol on next login
- [ ] Integration: update `max_file_size_mb` → attempt upload over new limit → verify rejection

## Acceptance Criteria
- Admin can perform full CRUD on users, departments, and document types
- Deactivated users cannot log in and their sessions are immediately revoked
- System settings take effect immediately without a server restart
- All admin actions are logged to `audit_logs`
- Department/document type deletion is blocked if records reference them

## Dependencies
- Task 1.4 (auth), Task 2.5 (RBAC), Task 2.1 (system_settings table)
