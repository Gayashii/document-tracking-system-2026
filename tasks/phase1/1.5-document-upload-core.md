# Task 1.5 — Document Upload (Core Flow)

**Phase:** 1 — MVP
**Status:** [x] Complete

## Objective
Implement secure document upload with reference number generation, local filesystem storage, and initial status tracking.

## Steps

### Storage Service (`storageService.js`)
- [x] Save uploaded files to `server/uploads/documents/{referenceNumber}/` (directory created on demand)
- [x] Expose: `save(fileBuffer, referenceNumber, originalFilename)` → returns relative file path; `delete(filePath)`
- [x] `server/uploads/` is git-ignored and must exist on the server before first run

### Reference Number Generator (`utils/referenceNumber.js`)
- [x] Use `sequences` DB table with `SELECT ... FOR UPDATE` lock to prevent race conditions
- [x] Format: `PGI-YYYY-NNNNNN` (year + 6-digit zero-padded sequence, resets each year)

### Upload Middleware (`middleware/uploadHandler.js`)
- [x] Configure `multer` with memory storage, 10 MB file size limit (env-configurable)
- [x] Validate MIME type by magic bytes using `file-type` package (not filename extension)
- [x] Allow only: `application/pdf`, `image/jpeg`, `image/png`
- [x] Reject anything else with 415 Unsupported Media Type

### Document Service (`documentService.js`)
- [x] `createDocument(file, metadata, actorId)`:
  1. Generate reference number (locked DB sequence)
  2. Save file to local filesystem via storage service; get back relative file path
  3. In a single DB transaction: insert `documents` row (with `file_path`) + insert `document_status_history` row (status = 'submitted') + insert `audit_logs` row
  4. Return created document with reference number

### Controllers & Routes (`document.routes.js`)
- [x] `POST /api/v1/documents` — multipart upload (auth required, roles: student, finance_staff, admin)
- [x] `GET /api/v1/documents` — list documents (scoped by role: students see only their own)
- [x] `GET /api/v1/documents/:id` — document detail
- [x] `GET /api/v1/documents/:id/file` — serve the file (auth required; streams file from `file_path` on disk)

### Joi Validators (`validators/document.validators.js`)
- [x] Upload schema: title (required), document_type_id (required), academic_year, semester, financial_amount
- [x] List query schema: page, limit, status, document_type_id

### Audit Logging
- [x] Log every upload: actor, action='document.upload', entity_type='document', entity_id, ip_address, user_agent

### Tests
- [ ] Unit: reference number generator (sequential, correct format, no duplicates under concurrent calls)
- [ ] Unit: MIME type validation rejects PDFs with wrong extension
- [ ] Integration: upload a PDF, verify it is saved to `server/uploads/`, verify DB row created with correct `file_path`, reference number, and status 'submitted'

## Implementation Notes
- `bcrypt` replaced with `bcryptjs` for cross-platform compatibility (no native bindings required)
- Added `GET /api/v1/lookups/document-types` and `/lookups/departments` endpoints (inline in `routes/index.js`) to serve dropdown data to the frontend
- Added `GET /api/v1/users` (admin-only, paginated) via `user.controller.js` + `user.routes.js`
- Added seed `03_document_types.js` with initial document types (idempotent — skips existing codes)
- `server/src/config/env.js`: `DB_PASSWORD` now allows empty string; schema set to `unknown(true)` to tolerate extra env vars

## Acceptance Criteria
- Uploading a valid PDF creates a document with a unique `PGI-YYYY-NNNNNN` reference number
- Uploading a file with wrong MIME type returns 415
- Files over 10 MB are rejected
- Student can only see their own documents in the list
- Authenticated file download endpoint streams the file correctly
