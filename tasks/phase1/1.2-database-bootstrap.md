# Task 1.2 — Database Bootstrap

**Phase:** 1 — MVP
**Status:** [x] Complete

## Objective
Stand up the core PostgreSQL schema using Knex.js migrations for the MVP tables.

## Steps

- [x] Install and configure Knex.js with `knexfile.js` (dev, test, production environments)
- [x] Create migration: `users` table
  - id, email (unique), password_hash, role ENUM('admin','finance_staff','student','auditor'), mfa_secret, mfa_enabled, is_active, created_at, updated_at, last_login_at
- [x] Create migration: `departments` table
  - id, name, code (unique), created_at
- [x] Create migration: `document_types` table
  - id, name, code (unique), description, requires_approval BOOL, created_at
- [x] Create migration: `documents` table
  - id, reference_number (unique), title, document_type_id (FK), student_id (FK → users), assigned_to (FK → users, nullable), status ENUM('submitted','pending_approval','approved','rejected','processed'), file_path, file_size, mime_type, academic_year, semester, financial_amount, version INT DEFAULT 1, created_at, updated_at, deleted_at
- [x] Create migration: `document_status_history` table
  - id, document_id (FK), from_status, to_status, changed_by (FK → users), note, ip_address, created_at
- [x] Create migration: `audit_logs` table (append-only)
  - id, actor_id (FK → users), action, entity_type, entity_id, metadata JSON, ip_address, user_agent, created_at
- [x] Create migration: `sequences` table for reference number generation
  - id, name (unique), current_value INT
- [x] Add critical indices: documents(status), documents(student_id), documents(created_at), documents(reference_number), audit_logs(actor_id, created_at)
- [x] Write dev seed files: `01_admin_user.js`, `02_departments.js`
- [x] No document type seed — types are created by the admin via the admin panel at runtime
- [x] Restrict DB user permissions: no UPDATE/DELETE on `audit_logs` table

## Acceptance Criteria
- `knex migrate:latest` runs with no errors on a fresh PostgreSQL 16 instance
- `knex seed:run` populates dev data including an admin user
- All foreign key constraints are enforced

## Dependencies
- Task 1.1 (Docker MySQL container must be running)
