# Task 1.6 — Frontend Skeleton (Angular)

**Phase:** 1 — MVP
**Status:** [x] Complete

## Objective
Build the Angular application structure with auth flows, role-based routing, and the document upload + list views.

## Steps

### Project Setup
- [x] Bootstrap with Angular CLI: `ng new client --routing --style=scss --standalone`
- [x] Install dependencies: `@angular/router`, `@angular/forms`, `@angular/common/http`, bootstrap, `ng-bootstrap`
- [x] Configure path aliases (`@/` → `src/app/`) in `tsconfig.json`
- [x] Configure `HttpClientModule` and `FormsModule`/`ReactiveFormsModule` in app config

### HTTP Client Layer (`src/app/core/services/api/`)
- [x] Create `auth.interceptor.ts`: attaches access token; on 401 → silently attempt token refresh → if refresh fails, clear auth state and redirect to `/login`
- [x] Create `auth-api.service.ts`, `document-api.service.ts` with typed methods for each endpoint

### Auth Service (`src/app/core/services/auth.service.ts`)
- [x] Store: `currentUser$` (BehaviorSubject: id, email, role), `accessToken`, `isLoading$`
- [x] Methods: `login(email, password)`, `logout()`, `refreshToken()`
- [x] Persist user info in `sessionStorage` (cleared on tab close)

### Routing (`src/app/app.routes.ts`)
- [x] `authGuard`: redirects to `/login` if no valid token
- [x] `roleGuard`: redirects to `/unauthorized` if user role not in allowed roles
- [x] Lazy-loaded route modules for: auth, documents, admin, reports

### Layouts
- [x] `AppLayoutComponent`: sidebar (role-aware nav links) + top bar (user info, logout, notification bell placeholder)
- [x] `AuthLayoutComponent`: centered card layout for login/register pages

### Auth Pages (`src/app/features/auth/`)
- [x] `LoginComponent`: email + password reactive form
- [x] `RegisterComponent`: student self-registration form
- [x] `ForgotPasswordComponent` + `ResetPasswordComponent`
- [~] `MfaVerifyComponent`, `MfaSetupComponent` — **dropped** (MFA removed from scope)

### Document Pages (`src/app/features/documents/`)
- [x] `DocumentListComponent`: paginated table with columns: Reference No, Title, Type, Status, Date; filter by status
- [x] `DocumentUploadComponent`: file picker (PDF/JPEG/PNG only, 10 MB limit enforced client-side), metadata reactive form, submit; document types loaded from `/lookups/document-types`
- [x] `DocumentDetailComponent`: metadata display, status badge, download button (blob download via HttpClient)

### Shared Components (`src/app/shared/components/`)
- [x] `StatusBadgeComponent`: colour-coded badge per document status
- [x] `FileUploadComponent`: drag-and-drop + click upload with client-side type/size validation
- [x] `SpinnerComponent`, `ModalComponent`, `PaginationComponent`

### Dashboard (`src/app/features/dashboard/DashboardComponent`)
- [x] Role-aware summary cards:
  - Student: "My Documents" count by status
  - Finance Staff: "Pending Review" count, "Recently Updated" list
  - Admin: total users, total documents, documents by status

### Tests
- [ ] Unit: `authGuard` redirects unauthenticated users
- [ ] Unit: `roleGuard` blocks wrong roles
- [ ] Unit: auth interceptor retries on 401 then redirects on second failure

## Implementation Notes
- MFA dropped from scope — `MfaVerifyComponent` and `MfaSetupComponent` stubs exist but are not wired
- API services unwrap the server's `{ success, data }` envelope; `mapDoc()` normalises snake_case fields from the DB to camelCase `DocumentMeta`
- File download uses `HttpClient` blob response + programmatic `<a>` click (avoids auth header issues with `window.open`)
- Auth service reads `full_name` or `fullName` from the login response (server returns snake_case)

## Acceptance Criteria
- Student can register, log in (with/without MFA), upload a document, and see it in their list with status "Submitted"
- Finance Staff sees all documents in their list
- Admin sees all documents and all users
- Unauthorized role access redirects cleanly with a message
- No console errors in production build
