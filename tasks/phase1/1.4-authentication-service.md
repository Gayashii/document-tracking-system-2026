# Task 1.4 — Authentication Service (JWT)

**Phase:** 1 — MVP
**Status:** [x] Complete

## Objective
Implement secure email/password login, registration, and JWT token lifecycle. No MFA.

## Steps

### Database Additions
- [x] Create migration: `refresh_tokens` table
  - id, user_id (FK), token_hash, expires_at, created_at, revoked_at
- [x] Create migration: `password_reset_tokens` table
  - id, user_id (FK), token_hash, expires_at, used_at
- [x] Create migration: drop `mfa_secret` and `mfa_enabled` columns from `users`

### Auth Service (`authService.js`)
- [x] Register: hash password with bcrypt (cost factor 12), insert user, return nothing sensitive
- [x] Login: validate credentials → issue access token (15-min JWT) + refresh token (7-day, stored as hash in DB, sent as HttpOnly cookie)
- [x] Refresh: validate refresh token from HttpOnly cookie → rotate (delete old, insert new) → issue new access token
- [x] Logout: revoke refresh token (set `revoked_at`)
- [x] Password reset request: generate time-limited token (1 hour), store hash in `password_reset_tokens`, send email
- [x] Password reset confirm: validate token, set new password, revoke all refresh tokens for that user

### Middleware
- [x] Create `middleware/authenticate.js`: verify JWT, attach `req.user` (id, email, role)
- [x] Create `middleware/authorize.js`: role-check factory `authorize(['admin', 'finance_staff'])`

### Controllers & Routes (`auth.routes.js`)
- [x] `POST /api/v1/auth/register`
- [x] `POST /api/v1/auth/login`
- [x] `POST /api/v1/auth/refresh`
- [x] `POST /api/v1/auth/logout`
- [x] `POST /api/v1/auth/forgot-password`
- [x] `POST /api/v1/auth/reset-password`

### Joi Validators
- [x] Register schema: email, password (min 8, complexity rules), role (optional, admin-only)
- [x] Login schema: email, password
- [x] Password reset schema: token, newPassword

### Tests
- [x] Unit tests: `authService` — password hashing, token generation
- [x] Integration tests: register → login → refresh → logout flow; password reset flow

## Acceptance Criteria
- Full auth flow works end-to-end
- Expired tokens return 401; revoked refresh tokens cannot be reused
- Password reset invalidates all existing sessions
- Brute-force blocked at 10 attempts/15min/IP on login endpoint

## Dependencies
- Task 1.2 (users, refresh_tokens, password_reset_tokens tables), Task 1.3 (middleware scaffold)
