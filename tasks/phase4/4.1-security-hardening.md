# Task 4.1 — Security Hardening

**Phase:** 4 — Production Readiness
**Status:** [ ] Not Started

## Objective
Audit and harden all security controls across the stack before production deployment.

## Steps

### Input Validation Audit
- [ ] Verify every API endpoint has a Joi schema validator attached via `validate` middleware
- [ ] Verify no endpoint accepts raw user input without validation
- [ ] Add missing validators for any endpoints added in Phase 2 or 3

### SQL Injection Prevention
- [ ] Audit all Knex queries: zero string interpolation in query builders
- [ ] Add ESLint rule (custom or `eslint-plugin-security`) to flag template literals in DB query context
- [ ] Run `npm audit` in both `client/` and `server/` — fix all high-severity vulnerabilities

### XSS Prevention
- [ ] Verify Angular templates escape all user-supplied data (no `[innerHTML]` binding without `DomSanitizer`)
- [ ] Verify `helmet` Content-Security-Policy is set and blocks inline scripts
- [ ] Review all email templates: ensure no user input is inserted as raw HTML

### CSRF Protection
- [ ] Verify access token is sent in `Authorization` header (not cookie) — structural CSRF mitigation
- [ ] Verify refresh token cookie is `SameSite=Strict; HttpOnly; Secure`
- [ ] Verify no state-changing endpoints accept GET requests

### File Upload Security
- [ ] Re-verify MIME type validation uses magic bytes (`file-type` package), not only extension
- [ ] Verify `server/uploads/` is not served as a static directory — files are only accessible via authenticated API endpoints
- [ ] Verify no uploaded file is executed or passed to a shell command
- [ ] Consider ClamAV integration via `clamscan` npm package (optional, if budget allows)

### Rate Limiting Audit
- [ ] Auth endpoints: 10 req/15min/IP
- [ ] General API: 100 req/min/IP
- [ ] Export endpoints: 5 req/min/IP
- [ ] Public scan endpoint: 20 req/min/IP
- [ ] Verify rate limiter uses IP from `X-Forwarded-For` correctly (trust proxy setting)

### Session Security
- [ ] Access token lifetime: 15 minutes
- [ ] Refresh token lifetime: 7 days with rotation
- [ ] Force logout on password change (revoke all refresh tokens)
- [ ] MFA enforcement: verify `system_settings.mfa_required_roles` is checked on login
- [ ] Session timeout: verify inactive sessions are not extended without interaction

### Secrets Management
- [ ] Audit `.env.example` — confirm no default secret values are committed
- [ ] Verify `JWT_SECRET` is at least 32 characters, randomly generated
- [ ] Document secret rotation procedure in `README.md`

### Security Headers Checklist
- [ ] `Strict-Transport-Security: max-age=31536000; includeSubDomains`
- [ ] `X-Content-Type-Options: nosniff`
- [ ] `X-Frame-Options: DENY`
- [ ] `Content-Security-Policy: default-src 'self'`
- [ ] `Referrer-Policy: strict-origin-when-cross-origin`
- [ ] `Permissions-Policy: camera=(), microphone=(), geolocation=()`

### Penetration Testing
- [ ] Run OWASP ZAP against staging environment
- [ ] Address all high and medium severity findings before go-live
- [ ] Document findings and resolutions

## Acceptance Criteria
- `npm audit` returns zero high-severity vulnerabilities
- OWASP ZAP scan returns zero high-severity findings on staging
- All security headers present on every API response
- MFA is enforced for admin and finance_staff roles (no bypass path)
- No user input reaches a DB query without Joi validation

## Dependencies
- All Phase 1–3 tasks complete
