# Task 4.2 — Encryption at Rest

**Phase:** 4 — Production Readiness
**Status:** [ ] Not Started

## Objective
Ensure sensitive database fields are protected at the application level in the production environment.

## Steps

### Application-Level Field Encryption (`utils/encryption.js`)
- [ ] Implement AES-256-GCM encryption/decryption helpers:
  - `encrypt(plaintext, key)` → returns `{ iv, ciphertext, tag }` as base64 string
  - `decrypt(encryptedString, key)` → returns plaintext
- [ ] Key: read from `FIELD_ENCRYPTION_KEY` env var (32-byte hex string)
- [ ] Apply to: `users.mfa_secret`
- [ ] Do NOT apply to indexed fields (encrypted data cannot be searched/indexed)

### Key Management
- [ ] Encryption keys stored in environment variables (never committed to code or DB)
- [ ] Document key rotation procedure: new key → re-encrypt affected fields → update env var

### Tests
- [ ] Unit: `encrypt(decrypt(x)) === x` for various input lengths
- [ ] Unit: changing IV produces different ciphertext for same plaintext
- [ ] Unit: decryption with wrong key throws `DecryptionError`

## Acceptance Criteria
- `users.mfa_secret` is encrypted at rest and decryptable only with `FIELD_ENCRYPTION_KEY`

## Dependencies
- Task 1.5 (storage service), Task 4.5 (production infrastructure)
