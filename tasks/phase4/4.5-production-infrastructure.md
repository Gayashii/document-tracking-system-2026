# Task 4.5 — Production Infrastructure

**Phase:** 4 — Production Readiness
**Status:** [ ] Not Started

## Objective
Configure the production server environment: Nginx reverse proxy, SSL, process supervision, structured logging, and monitoring.

## Steps

### Nginx Configuration (`nginx/nginx.conf`)
- [ ] SSL termination: configure with Let's Encrypt certificate (Certbot)
- [ ] Redirect all HTTP (port 80) to HTTPS (port 443)
- [ ] Proxy pass all `/api/` requests to Node.js server (port 3000)
- [ ] Serve React static bundle directly from Nginx (`/usr/share/nginx/html`)
- [ ] Enable gzip compression: `gzip on; gzip_min_length 1000; gzip_types text/plain application/json application/javascript text/css;`
- [ ] Set security headers in Nginx (HSTS, X-Frame-Options, X-Content-Type-Options)
- [ ] Client-side routing support: `try_files $uri $uri/ /index.html;` for React Router

### SSL Certificate
- [ ] Install Certbot on production server
- [ ] Obtain Let's Encrypt certificate for the production domain
- [ ] Configure auto-renewal cron job: `0 0 * * * certbot renew --quiet`
- [ ] Enable HSTS: `Strict-Transport-Security: max-age=31536000; includeSubDomains`

### Process Management
- [ ] Node.js runs inside Docker container with `restart: unless-stopped` policy
- [ ] Health check in `docker-compose.prod.yml`:
  ```
  healthcheck:
    test: ["CMD", "curl", "-f", "http://localhost:3000/api/health"]
    interval: 30s
    timeout: 10s
    retries: 3
  ```
- [ ] Graceful shutdown: Node server listens for SIGTERM, finishes in-flight requests, closes DB connections, exits cleanly

### Structured Logging (production)
- [ ] Configure `winston` with JSON format in production (not pretty-print)
- [ ] Log levels: `error` and `warn` always; `info` for request logs; `debug` only in dev
- [ ] Log to stdout (Docker captures it via logging driver)
- [ ] Optionally ship logs to: CloudWatch (AWS), Loki (self-hosted Grafana stack), or Papertrail

### Monitoring and Alerting
- [ ] Uptime monitoring: configure UptimeRobot or Better Uptime to poll `/api/health` every 5 minutes
- [ ] Alert channels: email + SMS on downtime
- [ ] Log monitoring: alert on error log rate spike (> 10 errors/minute)

### Environment Variables (production)
- [ ] `NODE_ENV=production`
- [ ] `DATABASE_URL` — production PostgreSQL connection string (e.g. `postgresql://user:pass@host:5432/doc_tracking`)
- [ ] `JWT_SECRET` — 32+ random bytes (generated with `openssl rand -hex 32`)
- [ ] `FIELD_ENCRYPTION_KEY` — 32-byte hex key for field-level encryption
- [ ] `UPLOAD_DIR` — absolute path to the uploads directory on the server (e.g. `/app/uploads`)
- [ ] `SMTP_HOST`, `SMTP_PORT`, `SMTP_USER`, `SMTP_PASS`, `EMAIL_FROM`
- [ ] `REDIS_URL` — production Redis connection string
- [ ] `ALLOWED_ORIGINS` — production frontend domain for CORS

## Acceptance Criteria
- HTTPS enforced on all connections; HTTP redirects to HTTPS
- SSL certificate auto-renews without manual intervention
- Node server restarts automatically on crash
- `/api/health` monitored every 5 minutes with alert on failure

## Dependencies
- Task 4.4 (Docker images), Task 4.2 (encryption at rest)
