# Task 4.3 — Performance Optimisation

**Phase:** 4 — Production Readiness
**Status:** [ ] Not Started

## Objective
Tune backend query performance, add caching, and optimise the frontend bundle to meet the <2 second response time target.

## Steps

### Database Index Audit
- [ ] Verify all indices from Task 1.2 are present in production
- [ ] Add any missing indices identified during Phase 2–3 development:
  - `notifications(recipient_id, status)` for unread count queries
  - `document_versions(document_id, version_number)`
  - `audit_logs(entity_type, entity_id)` for entity history lookups
- [ ] Run `EXPLAIN ANALYZE` on the 5 most frequent queries; add indices where `Seq Scan` appears on large tables
- [ ] Ensure connection pool is tuned: `pool: { min: 2, max: 10 }` in `knexfile.js`

### Caching (`config/redis.js` + `services/cacheService.js`)
- [ ] Add Redis to `docker-compose.yml` (dev) and production infrastructure
- [ ] Install `ioredis`
- [ ] `cacheService.js`: `get(key)`, `set(key, value, ttlSeconds)`, `del(key)`, `flush(pattern)`
- [ ] Cache targets:
  - Report results (`/reports/statistics`): TTL 60 seconds
  - System settings: TTL 300 seconds, invalidated on `PATCH /settings`
  - Document type and department lists: TTL 600 seconds, invalidated on admin CRUD
- [ ] Graceful degradation: if Redis is unavailable, log warning and skip cache (no crash)

### Backend Query Optimisation
- [ ] Document list query: use `SELECT id, reference_number, title, status, created_at` — avoid `SELECT *` to reduce data transfer
- [ ] Audit log query: add `LIMIT` enforcement (max 500 rows per page) to prevent massive result sets
- [ ] Report queries: test with 50,000 document rows; use `pg_stat_statements` extension to measure true query time

### Frontend Bundle Optimisation
- [ ] Run `source-map-explorer dist/client/browser/*.js` after production build
- [ ] Identify and remove any duplicate dependencies
- [ ] Verify Angular lazy-loaded routes are active for all feature modules (from Task 3.5)
- [ ] Check Bootstrap: import only used CSS via custom `_variables.scss` partial, not the full bundle
- [ ] Target: initial bundle ≤ 200 KB gzipped; per-route chunks ≤ 100 KB gzipped each
- [ ] Enable Angular CLI production build (`ng build --configuration production`) with `optimization: true` (minification + tree shaking by default)

### Image/Document Preview Optimisation
- [ ] Generate thumbnails server-side for uploaded images (JPEG/PNG) using `sharp`
  - Thumbnail: 300×300px, quality 70, stored as `server/uploads/thumbnails/{documentId}.jpg`
  - Generated asynchronously after upload (non-blocking)
- [ ] Document list shows thumbnail instead of full image
- [ ] Full image/PDF served via authenticated file endpoint on document detail page

### Response Compression
- [ ] Verify `compression` middleware is active in Express for all JSON API responses
- [ ] Verify Nginx is configured with `gzip on; gzip_types text/plain application/json;`

### Load Testing
- [ ] Run `autocannon` or `k6` load test against staging simulating 50 concurrent users
- [ ] Target: p95 response time < 2 seconds for document list, search, and status change endpoints
- [ ] Fix any endpoint that fails the target

## Acceptance Criteria
- All common queries have covering indices (no full table scans on tables with >1,000 rows)
- Document list loads in < 500ms (p95) with 10,000 documents in DB
- Initial Angular bundle is ≤ 200 KB gzipped
- Statistics report cache hit rate > 80% (observable via Redis `INFO stats`)
- Load test: p95 < 2 seconds at 50 concurrent users

## Dependencies
- All Phase 1–3 tasks complete; Task 4.5 (Redis in production infra)
