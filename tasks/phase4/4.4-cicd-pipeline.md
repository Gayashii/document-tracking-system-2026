# Task 4.4 — CI/CD Pipeline Finalisation

**Phase:** 4 — Production Readiness
**Status:** [ ] Not Started

## Objective
Finalize automated CI/CD pipelines covering lint, test, build, Docker image publishing, and staged deployment with a manual approval gate.

## Steps

### CI Workflow (`.github/workflows/ci.yml`)
- [ ] Trigger: on every pull request and push to `main`
- [ ] Steps in order:
  1. Checkout code
  2. Set up Node.js 20 LTS
  3. `npm ci` in both `client/` and `server/` (cached)
  4. ESLint (fail on any error)
  5. `npm audit` — fail on high-severity vulnerabilities
  6. Start Docker services: PostgreSQL 16 (via `docker-compose -f docker-compose.test.yml up -d`)
  7. Run `knex migrate:latest` against test DB
  8. Run `knex seed:run` for test seed data
  9. Run unit tests: `npm test -- --coverage` in `server/`
  10. Run integration tests: `npm run test:integration` in `server/`
  11. Build Angular app: `ng build --configuration production` in `client/`
  12. Upload coverage report as workflow artifact
  13. Tear down Docker services
- [ ] Block PR merge if any step fails

### Docker Build Workflow (`.github/workflows/deploy.yml`)
- [ ] Trigger: on push to `main` after CI passes
- [ ] Steps:
  1. Build Docker image for `server/`: `node:20-alpine` base, non-root user, only prod dependencies
  2. Build Docker image for `client/`: multi-stage — build in `node:20` → serve static files via `nginx:alpine`
  3. Tag images with Git SHA and `latest`
  4. Push to container registry (GitHub Container Registry: `ghcr.io`)
  5. Deploy to staging: SSH into staging server, `docker-compose pull && docker-compose up -d --no-deps`
  6. Run smoke tests against staging: `GET /api/health` returns 200, login endpoint reachable
  7. **Manual approval gate**: GitHub Environment protection rule on `production` environment
  8. On approval: deploy to production server (same pull + up pattern)
  9. Run post-deploy health check on production

### Docker Configuration
- [ ] `server/Dockerfile`:
  ```
  FROM node:20-alpine
  RUN addgroup -S app && adduser -S app -G app
  WORKDIR /app
  COPY package*.json ./
  RUN npm ci --omit=dev
  COPY src/ ./src/
  USER app
  EXPOSE 3000
  CMD ["node", "src/server.js"]
  ```
- [ ] `client/Dockerfile`: multi-stage — build Angular app in `node:20` stage → copy `dist/client/browser/` to `nginx:alpine` stage
- [ ] `docker-compose.prod.yml`: production overrides (no dev ports exposed; `uploads/` volume mounted)

### Smoke Tests (`scripts/smoke-test.sh`)
- [ ] `GET /api/health` → 200 with `{ status: 'ok', db: 'ok', storage: 'ok' }`
- [ ] `POST /api/v1/auth/login` with invalid credentials → 401
- [ ] `GET /api/v1/documents` without auth → 401
- [ ] Frontend static file served: `GET /` → 200 with correct HTML title

### Test Coverage Requirements
- [ ] Set coverage thresholds in Jest config: statements ≥ 70%, branches ≥ 65%, functions ≥ 70%
- [ ] CI fails if coverage drops below threshold

### Dependency Updates
- [ ] Add `dependabot.yml` to auto-create PRs for dependency updates (weekly, grouped by ecosystem)
- [ ] Security updates: immediate PRs (Dependabot default)

## Acceptance Criteria
- CI runs on every PR and blocks merge on any failure
- Docker images are automatically built and pushed on merge to `main`
- Staging deploys automatically; production requires manual approval
- Smoke tests verify basic system health post-deploy
- Coverage thresholds enforced in CI
