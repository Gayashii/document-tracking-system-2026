# Task 2.2 — Workflow Engine

**Phase:** 2 — Core Features
**Status:** [x] Complete

## Objective
Implement a state machine for document status transitions with role guards, atomic DB operations, and event emission for notifications.

## Steps

### Transition Map (`constants/transitions.js`)
- [x] Define allowed transitions as a map:
  ```
  submitted        → pending_approval  (allowed roles: finance_staff, admin)
  submitted        → rejected          (allowed roles: finance_staff, admin)
  pending_approval → approved          (allowed roles: finance_staff, admin)
  pending_approval → rejected          (allowed roles: finance_staff, admin)
  approved         → processed         (allowed roles: finance_staff, admin)
  rejected         → submitted         (allowed roles: student)  ← re-submission
  ```
- [x] Each transition entry includes: `to`, `allowedRoles[]`, `label`

### Workflow Service (`services/workflowService.js`)
- [x] `transition({ documentId, toStatus, actor, note, ipAddress, userAgent })`:
  1. Fetch document current status from DB
  2. Look up allowed transition in `transitions.js` — throw `WorkflowError` (422) if invalid
  3. Check actor's role is in `allowedRoles` — throw `ForbiddenError` (403) if not
  4. Within a single DB transaction:
     - Update `documents.status` + `updated_at`
     - Insert row to `document_status_history`
     - Insert row to `audit_logs`
  5. Emit `document.status_changed` event via singleton `events/documentEvents.js` (Node.js EventEmitter)
  6. Return updated document
- [x] `getHistory(documentId)` — returns status history joined with changer's user info

### Error Types (`utils/errors.js`)
- [x] `WorkflowError` (HTTP 422) — already existed
- [x] `ForbiddenError` (HTTP 403) — already existed
- [x] `NotFoundError` (HTTP 404) — already existed

## Acceptance Criteria
- Every status change is atomic (document row + history row + audit log in one transaction)
- Invalid transitions return 422 with a clear error message
- Role violations return 403
- `document.status_changed` event is emitted after every successful transition
