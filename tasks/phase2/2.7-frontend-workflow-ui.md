# Task 2.7 — Frontend Document Workflow UI

**Phase:** 2 — Core Features
**Status:** [x] Complete

## Objective
Build the document workflow UI: status timeline, action buttons per role, finance staff queue, and student tracker.

## Steps

### Status Timeline Component (`src/app/shared/components/status-timeline/`)
- [x] Visual CSS stepper showing all 5 statuses in order: Submitted → Pending Approval → Approved / Rejected → Processed
- [x] Highlight current status; show completed steps; show rejection branch if applicable
- [x] Below each completed step: show timestamp, actor name, and note from `document_status_history`
- [x] No external library — pure CSS with Bootstrap utilities

### Document Detail Page Update (`DocumentDetailComponent`)
- [x] Display `StatusTimelineComponent`
- [x] Role-aware action buttons:
  - **Finance Staff / Admin**: "Mark as Pending Review", "Approve", "Reject" (with required note field on reject)
  - **Student** (on rejected doc): "Re-submit" button → opens file upload modal
  - **Auditor**: no action buttons
- [x] Status change modal: confirm dialog with optional note field; calls `PATCH /documents/:id/status`
- [x] On success: optimistic status update via BehaviorSubject; revert on API error with toast

### Finance Staff Queue (`DocumentListComponent` update)
- [x] Default filter for finance staff: show `pending_approval` documents first
- [x] Quick-action column: "Approve" / "Reject" directly from list (opens inline confirm)
- [x] Sort by: oldest first (longest wait), submission date, document type
- [x] Filter by: status, document type, date range, student name/ID

### Student Document Tracker
- [x] Student dashboard: list of own documents grouped by status
- [x] Each row: reference number, title, status badge, last updated, "View" link
- [x] Clear call-to-action for rejected documents: "Action Required — Re-submit"

### Re-submission Flow
- [x] Modal in `DocumentDetailComponent` for re-submission: file picker + change note textarea
- [x] Calls `POST /documents/:id/versions`
- [x] On success: status badge updates to "Submitted", re-submit button disappears

### Toast Notification Service (`src/app/core/services/toast.service.ts`)
- [x] Global toast service: `show(message, type)` where type is `success|error|info|warning`
- [x] `ToastContainerComponent`: fixed-position overlay (top-right), auto-dismiss after 4 seconds
- [x] No external library — pure Angular + CSS

### Tests
- [x] Unit: `StatusTimelineComponent` renders correct step as active for each status value
- [x] Unit: action buttons render correctly per role (finance_staff sees approve/reject, student sees re-submit only on rejected)
- [x] Unit: optimistic update reverts on API error

## Acceptance Criteria
- Finance staff can approve or reject documents from both the list and detail views
- Students see a clear status timeline and "Re-submit" CTA when rejected
- Status changes reflect immediately in the UI without a full page refresh
- Auditors see the full timeline but no action buttons

## Dependencies
- Task 2.2 (status endpoints), Task 2.5 (RBAC enforced), Task 2.6 (re-submission endpoint), Task 1.6 (frontend scaffold)
