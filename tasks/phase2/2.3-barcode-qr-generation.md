# Task 2.3 — Barcode Number (Manual Entry)

**Phase:** 2 — Core Features
**Status:** [x] Complete

## Objective
Allow finance staff to manually enter a unique barcode number on a document record for physical document tracking and quick lookup.

## Steps

### Database
- [x] `barcode_number` column (unique, nullable, max 100) already present from task 1.2 migration — no new migration needed

### Backend
- [x] `PATCH /api/v1/documents/:id/barcode` — set or update `barcode_number` (auth: finance_staff, admin)
  - Validates non-empty string, max 50 characters via `barcodeSchema`
  - Pre-checks uniqueness and throws `ConflictError` (409) before hitting the DB constraint
  - Logs to `audit_logs` (action: `DOCUMENT_BARCODE_SET`)
- [x] `GET /api/v1/scan?code=<value>` — public lookup, no auth, rate-limited 20 req/min/IP (`scanLimiter`)
  - Returns: reference_number, title, status, updated_at, document_type — nothing sensitive

### Frontend
- [x] `DocumentMeta` interface extended with `barcodeNumber: string | null`
- [x] `DocumentApiService.setBarcode(id, code)` method added
- [x] `DocumentDetailComponent`: inline edit (Assign/Edit button + input + Save/Cancel) for finance_staff and admin; read-only display for other roles

### Tests
- [x] Unit: successful barcode assignment returns updated document
- [x] Unit: duplicate barcode number throws `ConflictError` (409)
- [x] Unit: document not found throws `NotFoundError` (404)

## Acceptance Criteria
- Finance staff can assign a barcode number to any document
- Duplicate barcode numbers are rejected
- Public scan endpoint returns document status without requiring login
- Barcode number is searchable in the document list

## Amendment — Numeric Barcode (Code 128)
Auto-generated barcodes changed from 12-char alphanumeric (A-Z0-9) to **12-digit numeric** for
compatibility with Code 128 physical barcode printers and USB/Bluetooth readers.

- `server/src/utils/barcode.js` — generator now produces digits 0–9 only
- `server/src/validators/document.validators.js` — `barcodeSchema` now enforces `^\d+$` (digits only, max 20)
- Existing alphanumeric barcodes in the DB are left untouched; staff reassign via the edit UI

## Dependencies
- Task 1.2 (documents table), Task 1.4 (auth), Task 2.5 (RBAC)
