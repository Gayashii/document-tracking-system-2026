# Task 2.4 — Notification Service

**Phase:** 2 — Core Features
**Status:** [x] Complete

## Objective
Send automated email and in-app notifications on document status changes using an event-driven pattern.

## Steps

### Dependencies
- [x] `nodemailer` already in `server/package.json`

### Email Templates (`templates/email/`)
- [x] `documentSubmitted.html`
- [x] `statusChanged.html` — with `{{#note}}...{{/note}}` conditional block
- [x] `documentApproved.html`
- [x] `documentRejected.html` — with `{{#note}}...{{/note}}` rejection reason block
- [x] `passwordReset.html` — with reset URL and 1-hour expiry notice
- [x] `renderTemplate(name, vars)` — replaces `{{key}}` tokens and handles `{{#key}}...{{/key}}` conditional blocks

### Mailer Config (`config/mailer.js`)
- [x] Updated to lazy-initialize transporter via `getTransporter()`
- [x] Ethereal.email fallback for dev/test when `SMTP_USER` not set; logs preview URL to console
- [x] `sendMail(options)` async function (backward-compatible interface)

### Notification Service (`services/notificationService.js`)
- [x] Registers listener on `documentEvents` singleton for `document.status_changed`
- [x] On event: fetches document + student email, picks template by `toStatus`, sends email
- [x] Inserts `email` notification row (`pending`) before send; updates to `sent` or `failed`
- [x] Retry logic: 3 attempts, exponential backoff (1s, 2s — delays before attempts 2 and 3)
- [x] Failed send: updates row to `failed`, logs to `audit_logs`
- [x] `sendEmail(to, subject, htmlBody)` exported for testing

### In-App Notifications
- [x] Inserts `in_app` notification (`status='pending'` = unread) after each status change
- [x] `GET /api/v1/notifications/mine` — paginated, pending first
- [x] `GET /api/v1/notifications/unread-count` — lightweight poll endpoint for the bell
- [x] `PATCH /api/v1/notifications/:id/read` — sets status='sent', sent_at=now
- [x] `PATCH /api/v1/notifications/read-all` — bulk mark all pending in_app as read

### Frontend (`src/app/shared/components/notifications/`)
- [x] `NotificationApiService` — getMine, getUnreadCount, markRead, markAllRead
- [x] `NotificationBellComponent` — badge with unread count, 60s polling via `interval` RxJS
- [x] `NotificationListComponent` — dropdown, mark single/all read, unread items highlighted
- [x] Bell wired into `AppLayoutComponent` topbar

### Tests
- [x] Unit: `renderTemplate` token substitution (5 cases across all templates)
- [x] Unit: `renderTemplate` conditional block rendered when truthy, omitted when falsy
- [x] Unit: `sendEmail` succeeds on first attempt
- [x] Unit: `sendEmail` retries on failure and succeeds on second attempt
- [x] Unit: `sendEmail` calls send exactly 3 times then throws
- [ ] Integration: trigger a status change, verify notification record created with status 'sent'

## Acceptance Criteria
- Student receives an email when their document status changes
- Email delivery failures are logged and retried up to 3 times
- All failed notifications are visible in the `notifications` table for finance staff to check
- In-app notification bell shows unread count without page refresh (polling)
