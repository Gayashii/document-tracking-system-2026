# Task 2.3b — Barcode Image Scan

**Phase:** 2 — Core Features
**Status:** [x] Complete

## Objective

Allow finance staff to upload a photo/image of a physical document's barcode instead of manually typing the code. The system reads the barcode from the image client-side and navigates directly to the matching document — identical outcome to typing the code by hand.

## Approach

Client-side decoding via `@zxing/library`. The decoded barcode string is passed to the existing `GET /api/v1/scan?code=` endpoint unchanged. No backend changes are required.

## Steps

### Dependency

- [ ] Install `@zxing/library` in the client package
  ```bash
  cd client && npm install @zxing/library
  ```

### Frontend — `document-list.ts`

- [ ] Add `ElementRef` and `viewChild` to the Angular core import
- [ ] Import `BrowserMultiFormatReader` from `@zxing/library`
- [ ] Add `barcodeImageDecoding = signal(false)` signal
- [ ] Add `barcodeFileInput = viewChild<ElementRef<HTMLInputElement>>('barcodeFileInput')` view child reference
- [ ] Add `openBarcodeImagePicker()` method — triggers a `.click()` on the hidden file input
- [ ] Add `async onBarcodeImageSelected(event: Event)` method:
  - Reads the selected `File` from the input event
  - Creates a temporary object URL (`URL.createObjectURL`)
  - Resets the file input value immediately (so the same file can be re-selected)
  - Sets `barcodeImageDecoding` to `true`
  - Calls `new BrowserMultiFormatReader().decodeFromImageUrl(url)`
  - On success: sets `barcodeInput` to `result.getText()` and calls `onBarcodeScan()`
  - On failure: shows toast `"Could not read a barcode from this image. Try a clearer photo."`
  - Always revokes the object URL and sets `barcodeImageDecoding` back to `false`

### Frontend — `document-list.html`

- [ ] Add a hidden file input inside the barcode card block:
  ```html
  <input type="file" #barcodeFileInput accept="image/*" style="display:none"
    (change)="onBarcodeImageSelected($event)" />
  ```
- [ ] Add an image-upload icon button to the barcode input-group (between the text input and the "Go" button):
  - Disabled when `barcodeScanning()` or `barcodeImageDecoding()`
  - Shows a spinner while `barcodeImageDecoding()`
  - Uses a photo/image SVG icon when idle
  - `title="Upload barcode image"` for accessibility
- [ ] Add `barcodeImageDecoding()` to the disabled condition of the existing "Go" button

## Acceptance Criteria

- Clicking the image icon opens a file picker (images only)
- A valid barcode image navigates to the correct document (same as typing the code)
- An image with no readable barcode shows an error toast; input is left unchanged
- The image button shows a spinner while decoding is in progress
- Both the image button and the "Go" button are disabled while a scan or decode is in progress
- No changes to the backend are required
